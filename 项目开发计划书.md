# 问窟 GrottoMind — 项目开发计划书

> 让沉默的石窟重新被看见、被理解、被参与。

---

## 一、项目概述

### 定位

一个面向公众与设计观众的**栖霞山石窟造像 AI 数字复彩交互档案馆**。不是普通文化介绍网站，而是一次用数字技术重新"进入"石窟的体验。

### 核心立场

> 本项目中的"复彩"并非对历史原貌的绝对复原，而是基于文化资料、视觉研究与 AI 技术的**数字化色彩推演**。我们不是要把石窟修成新的，而是让观众看见：石窟曾经的色彩可能性、今日的残损状态、未来的数字传播方式。

### 审美标杆

[Getty Persepolis Reimagined](https://persepolis.getty.edu/) — 博物馆级叙事、电影感过渡、呼吸感节奏。

---

## 二、叙事结构：一次进入石窟的旅程

整个网站是一次从"远望山门"到"触摸岩壁"的叙事旅程。用户不是在"浏览网站"，而是在"走进石窟"。

```
序章 · 远望        →  山门之外，雾气弥漫，舍利塔隐约可见
第一章 · 舍利塔    →  推近塔身，观看层檐、佛龛与石上纹理
第二章 · 看见褪色  →  走近岩壁，看见风化留下的纹理
第三章 · 复彩实验室 →  亲手参与色彩的显影
第四章 · 问窟      →  向石窟提问，AI 作为岩壁中的声音回应
第五章 · 共创一龛  →  留下属于自己的栖霞色彩记忆
尾声 · 作品说明    →  幕后解说
```

### 第一章 · 舍利塔 — 局部漫游

**情绪**：静穆、近距离、带有考古式凝视。用户不是进入一个普通 3D 展厅，而是被带到舍利塔前，在云雾与暗金光点之间逐步接近塔身细节。

**核心创意**：
第一章只围绕 **舍利塔** 展开，不再泛化为山、寺、窟的综合介绍。交互逻辑参考 Getty Persepolis 的局部观看方式：用户轻微滚动，镜头便从舍利塔整体推向不同局部；当镜头抵达某一局部时，画面中浮现一个带古典纹样边缘的圆形“放大镜”热点。点击热点后，全屏预览该局部细节图与对应说明。

**视觉与交互方案**：

1. **主体场景**：
   - **主体**：画面中心为栖霞山舍利塔高精度 OBJ 模型，经过重新居中、上移与轴线校正，保证视觉中心稳定。
   - **环境**：模型背后放置低明度 360° 全景环境图，只在云雾后方轻微显现，避免抢夺舍利塔主体。
   - **氛围**：保留黑场、雾气、浮尘、暗金星点与弱光束，让暂停帧也具备呼吸感。

2. **滚动镜头**：
   - 用户滚轮不进入传统页面滚动，而是驱动 3D 相机沿预设轨迹移动。
   - 镜头路径分为四段：`塔身概览 → 层檐 → 塔身佛龛 → 石上纹理`。
   - 每一段都与舍利塔局部相关，文案随节点切换，不再出现与本章无关的 AI 或共创说明。

3. **局部放大镜热点**：
   - 当镜头接近层檐、佛龛、石纹等局部时，画面浮现圆形热点。
   - 热点外圈使用细密古典纹样，中心为局部材质预览，效果类似“放大镜”。
   - 热点只在对应镜头段落出现，不常驻，不堆叠。

4. **全屏细节预览**：
   - 点击热点后进入全屏细节图预览。
   - 预览层包括：大幅局部图、局部标题、说明文案、返回按钮。
   - 当前阶段可先使用模型贴图作为占位，后续替换为真实局部高清截图或重渲染细节图。

**第一章节点文案**：

```
舍利塔
1400 年前，金陵栖霞山立起一座承载信仰与时间的舍利塔。它不是孤立的建筑，而是山、寺、窟之间的精神坐标。

层檐
层层石檐向外舒展，承接天光，也留下风化的阴影。檐口的起伏让舍利塔在静止中仍有向上的节奏。

塔身佛龛
塔身龛像与浮雕把信仰刻入石面。人物、衣纹与龛壁在岁月中磨损，却仍保留可被重新观看的线索。

石上纹样
裂隙、磨蚀与残留纹理不是噪点，而是时间的细节。它们提示我们：数字显影应先尊重石头本身。
```

**技术实现路径**：

- **基础库**：当前阶段继续使用原生 `Three.js + OBJLoader`，避免过早引入 `@react-three/fiber` 与 `drei` 增加结构复杂度。
- **模型加载**：加载 `/assets/qixia-model/3DModel0.obj` 与 `/assets/qixia-model/3DModel.jpg`，后续可转为压缩 `.glb` 优化性能。
- **镜头绑定**：将内部漫游进度 `0-1` 映射为相机的角度、半径、高度与 lookAt 目标点。
- **环境交互**：鼠标拖动只轻微旋转背景环境，主相机始终 look at 舍利塔，避免用户把主体拖出画面。
- **热点实现**：热点第一版采用 DOM 覆盖层，保证视觉可控；后续如需更精确，可再用 3D 坐标投影绑定到模型局部。
- **细节预览**：点击热点打开全屏覆盖层，展示局部高清图与说明文案。

**素材需求**：

- **已具备**：舍利塔 OBJ 模型、模型贴图、临时 360° 环境图。
- **下一步建议补充**：层檐、佛龛、石纹三个局部的高清截图或重新渲染图。
- **可由 AI 生成占位**：暗色栖霞山 360° 全景、塔身细节气氛图、放大镜内的局部纹理图。

### 第二章 · 看见褪色 — 风化叙事

**情绪**：安静、略带伤感，像在博物馆的暗室里凝视一件残损的文物。

**视觉**：一尊造像图片占据全屏。用户滚动时，画面经历三个阶段：

1. **完整期**：饱和度正常，色彩温暖
2. **风化期**：饱和度下降，裂纹纹理逐渐叠加，画面泛灰
3. **显影期**：从灰暗中，局部色彩开始微微浮现——引出下一章"复彩实验室"

**文案**（随滚动逐句浮现）：

```
风化不是消失，
而是时间留下的纹理。

残缺不是空白，
而是历史留下的问题。

数字复彩的意义，
不是覆盖旧痕，
而是让观众重新理解
造像曾经可能拥有的视觉生命。
```

**技术**：一张高清图 + CSS `filter`（`saturate`/`sepia`/`brightness`）滚动驱动 + PNG 裂纹纹理叠加。和开场视频滤镜同理，已验证可行。

**素材需求**：1 张高清造像正面图 + 1 张裂纹纹理透明 PNG

### 第三章 · 复彩实验室 — 核心交互

> [!IMPORTANT]
> 这是整个网站的**视觉高潮**和**答辩核心展示**。

**情绪**：从沉静转向参与感，用户从"观看者"变为"操作者"。

#### 模块 A：复彩对比滑杆

全屏左右分割。左侧现状灰色岩壁，右侧数字复彩效果。中间可拖拽分割线。

**引导文案**：

```
拖动，观看色彩从岩壁中显影。
```

技术：两张图叠放 + CSS `clip-path` 控制，极其轻量。

#### 模块 B：色彩方案切换

6 个预设方案，每个对应一张预渲染图片：

| 方案     | 色调   | 创意说明                                             |
| -------- | ------ | ---------------------------------------------------- |
| 岩灰原貌 | 冷灰   | 忠实记录现状，不加任何色彩推演                       |
| 朱砂显影 | 暖红   | 佛教造像传统中的庄严之色，朱砂是古代壁画最常见的颜料 |
| 石青石绿 | 冷蓝绿 | 取自敦煌、云冈等石窟的矿物色传统，清冷而深远         |
| 暗金佛光 | 金色   | 佛像头光、背光的神圣光晕，沉静而不张扬               |
| 丹枫栖霞 | 红橙   | 融入栖霞山秋季红叶意象，地域记忆与佛教色彩的交融     |
| AI 推演  | 混合   | 用户输入关键词，调用后端 API 生成色彩方案建议        |

技术：图片预加载 + `opacity` 交叉淡入。AI 推演调用已有的 `/api/recolor-card`。

#### 模块 C：局部点击解释

造像图片上 5 个可点击热点，点击弹出石刻拓片风格的解释卡片：

| 热点 | 解释文案                                                                                                 |
| ---- | -------------------------------------------------------------------------------------------------------- |
| 头光 | 头光是佛像最重要的视觉中心。在数字复彩中，通过暗金色光晕强化其神圣感。此效果为视觉推演，非历史原貌判断。 |
| 面部 | 南朝造像面部多呈微笑，线条圆润。风化后五官模糊，复彩尝试通过明暗对比恢复其神情层次。                     |
| 衣纹 | 造像衣纹流畅，体现六朝"秀骨清像"风格。复彩中以浅赭石色强调衣褶的光影转折。                               |
| 佛龛 | 佛龛拱形轮廓是石窟建筑的重要特征。龛楣、龛柱上可能曾有彩绘装饰。                                         |
| 底座 | 莲座是佛教造像的标志性元素。在色彩推演中，莲瓣以石青、石绿为主色调。                                     |

技术：CSS 绝对定位按钮 + GSAP 弹出动画。

#### 模块 D：TouchDesigner 作品嵌入

**标题**：TouchDesigner 数字复彩动态实验

**说明文案**：

```
本部分基于 TouchDesigner 构建视觉交互系统，
通过粒子、光晕、色彩扩散与图像反馈，
模拟石窟造像色彩从岩壁中逐渐显影的过程。

TouchDesigner 作品是"看见色彩重生"的沉浸式现场，
网站是"理解色彩重生"的智能交互档案。
两者共同构成完整的数字复彩设计系统。
```

技术：全屏视频播放（使用已有的 `grotto-flight.mp4`）。

**素材需求**：造像原图 + 各色彩方案预渲染图（至少 5 张）+ 确认 TD 视频

### 第四章 · 问窟 AI — 智能体对话

**情绪**：安静的对话感，像在黑暗的石窟中听到一个克制而有学识的声音。

**智能体名称**：问窟者

**角色设定**：不是客服机器人，而是"岩壁中的声音"——融合数字策展人、复彩研究助手、虚拟导览者三重身份。

**性格**：沉静、克制、有文化感、不神秘化、能把复杂知识讲简单。

**视觉**：

- 暗色全屏，左侧岩壁纹理背景
- 右侧对话面板：深色毛玻璃 + 金色细描边
- 不用聊天气泡，用"石刻拓片"风格的问答卡片
- AI 回答逐字显现（打字机效果）

**预设问题**（降低使用门槛）：

```
栖霞山石窟造像有什么特别之处？
为什么要进行数字复彩？
数字复彩和文物修复有什么区别？
网站中的颜色是历史真实颜色吗？
AI 在这个项目中起什么作用？
```

**用户身份选择**：`普通游客 | 设计师 | 研究者 | 文保兴趣者`

同一问题根据身份给出不同深度回答。

**技术**：调用已有的 `/api/ask` 接口。前端用 `fetch` + `requestAnimationFrame` 逐字渲染。

**素材需求**：岩壁纹理背景图

## 七、Vibe Coding 工作流

每个展厅按四轮迭代推进：

```
第 1 轮 · 骨架 → "搭一个空壳，暗色背景，基本布局"
第 2 轮 · 内容 → "把文案和图片填进去，调整排版"
第 3 轮 · 动效 → "加滚动渐显，加过渡动画"
第 4 轮 · 打磨 → 根据截图微调细节
```

**关键原则**：

1. 一次只做一个展厅
2. 先看效果再调整
3. 用截图标注问题
