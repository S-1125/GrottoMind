# 《问窟 GrottoMind》设计规范说明书

## 1. 项目视觉定位

《问窟 GrottoMind》不是普通文旅网页，也不是功能型 AI 工具页面，而是一个具有电影感、展览感与空间感的沉浸式文化遗产交互网站。

整体气质应保持：

- 沉静
- 庄重
- 神秘但不玄学
- 东方文化感但不仿古
- 数字技术感但不赛博
- 像博物馆沉浸展，而不是旅游宣传页

参考网站为 [Persepolis Reimagined](https://persepolis.getty.edu/)。参考重点不是复制其具体内容，而是学习它的表达方式：

- 全屏空间场景
- 极少 UI
- 大字号电影字幕
- 慢节奏滚动叙事
- 右下角轻量控制按钮
- 背景画面承担主要叙事
- 文本只在关键节点出现

后续所有页面都应像“进入一个场景”，而不是“打开一个网页板块”。

## 2. Logo 使用规范

### 2.1 主 Logo

当前项目 Logo 由两部分组成：

- 中文：`logo.png`
- 英文：`logo-en.png`

当前页面中使用 CSS mask 的方式显示，颜色为纯白。

使用位置：

- 默认固定在左上角。
- 与参考站 Getty 左上角品牌位一致。
- 不进入普通导航栏样式。
- 不加背景框。
- 不放进卡片。
- 不使用彩色版本。

当前规格建议保持：

- 中文 Logo：约 `59px × 25px`
- 英文 Logo：约 `134px × 18px`
- Logo 间距：`14px`
- 顶部距离：`20px - 32px`
- 左侧距离：`20px - 40px`

### 2.2 Logo 显隐规则

- 序章第一段可显示完整 Logo。
- 进入后续叙事时，可以淡出 Logo，让画面更沉浸。
- 如果后续场景需要品牌回归，只能以轻微淡入方式出现。
- 不允许 Logo 跟随滚动剧烈移动、旋转、缩放。
- 不允许把 Logo 做成按钮式 UI 或装饰图案。

### 2.3 Logo 颜色

默认只能使用：

- 白色：用于暗色视频或深色场景。
- 暗墨色：仅在极亮背景上使用。
- 暗金色只可作为特殊状态点缀，不作为常规 Logo 色。

## 3. 字体规范

### 3.1 叙事大字

当前中央叙事文字使用：

`ZiXinFangMengHuanGeTeTi`

对应文件：

`/public/assets/ZiXinFangMengHuanGeTeTi-2.ttf`

使用范围：

- 开头页四句主文案
- 章节标题
- 关键仪式性文字
- 最终大标题 `GrottoMind / 问窟`

不适合用于：

- 长正文
- 按钮说明
- AI 回答正文
- 技术说明
- 信息密集区域

当前中央字幕规则应保持：

- 颜色：纯白或接近纯白。
- 位置：屏幕正中。
- 对齐：居中。
- 行数：最多两行。
- 文案长度：尽量短，不做长段解释。
- 阴影：允许柔和黑色阴影保证可读性。
- 动画：从下方轻微浮现，透明度从 0 到 1。

禁止：

- 字体随意替换。
- 大段文字都用这个字体。
- 使用廉价发光描边。
- 使用粗暴投影。
- 同屏出现两句主文案。

### 3.2 最后一屏大标题

最后一屏 `GrottoMind` 应作为视觉高潮处理。

规范：

- 字号可以远大于普通叙事字幕。
- 居中或略上居中。
- 保持白色。
- 可以配合轻微粒子、金色微光、雾气流动。
- 不要加副标题堆叠太多信息。

如果出现中文“问窟”，建议作为小型附属文字出现，而不是和英文争夺主视觉。

### 3.3 加载数字字体

加载数字当前方向是哥特/戏剧感字体。

推荐保持：

- 字体风格：`New Rocker` 类似黑体哥特数字。
- 颜色：`#f6cea0`。
- 使用范围：只用于 `0-100` 加载数字。
- 不扩展到正文或普通 UI。

如果后续本地化字体文件，建议统一放入：

`/public/assets/`

并在 CSS 中用 `@font-face` 明确注册，避免浏览器 fallback 导致视觉变化。

### 3.4 正文与说明文字

正文、按钮提示、AI 回答、小字说明建议使用现代无衬线：

- `Inter`
- `Noto Sans SC`
- `Helvetica Neue`
- `Arial`

正文规范：

- 字号克制。
- 行距宽松。
- 不居中堆长文。
- 不出现大面积说明文字。
- 文字作为“展签”，不是文章页面。

## 4. 色彩规范

### 4.1 主色

| 名称 | 色值 | 用途 |
| --- | --- | --- |
| 深岩灰 | `#252525` | 页面底色、加载背景 |
| 墨黑 | `#151412` | hover 后深色文字、亮底反色 |
| 暗金 | `#f6cea0` | 进度、边框、关键光感 |
| 金线 | `#d4a96a` | 细进度条、按钮描边 |
| 雾白 | `rgba(255, 255, 255, 0.94)` | 主要文字 |

### 4.2 色彩原则

- 整体低饱和。
- 金色只做“光”和“线”，不要大面积铺满。
- 朱砂、丹枫红只在复彩显影模块出现。
- 石青、石绿只能作为复彩方案或局部 AI 辅助色，不作为主背景。
- 不使用荧光蓝、荧光紫、赛博渐变。
- 不使用大面积米黄、棕橙，避免廉价古风感。

### 4.3 遮罩规范

背景视频上方必须允许有暗色遮罩。

用途：

- 保证白字可读。
- 统一画面质感。
- 制造展览暗场感。

遮罩不应过重到看不清视频主体，也不应频繁闪烁。

## 5. 画面层级规范

当前序章层级可以作为后续基准：

1. 背景视频 / 场景图
2. 视频滤镜层
3. 暗色遮罩
4. 暖色叠加 / 暗角
5. 雾气 / 光束 / 粒子
6. 胶片颗粒
7. 中央字幕
8. 底部引导 / 进入按钮
9. 左上角 Logo / 右下角控制按钮
10. 激光进度条

后续新增任何模块都必须遵守这个逻辑：

- 场景先行。
- 氛围覆盖。
- 文字浮现。
- UI 最少。
- 信息最后出现。

不要在画面上叠大量卡片、图标、标签。

## 6. 视频与场景规范

### 6.1 背景视频

背景视频是主视觉核心。

使用规则：

- 全屏固定。
- `object-fit: cover`。
- 无声、内联播放。
- 不显示原生控制条。
- 由滚动或状态控制播放进度。
- 优先使用 scrub 专用 MP4 或图片序列。

推荐视频规格：

- 1920×1080
- 24fps
- 时长 6-12 秒
- 关键帧密度高
- 色调稳定
- 镜头运动缓慢
- 不要快速剪辑

### 6.2 视频暂停问题

如果视频停在某一帧，画面不能“死掉”。

必须叠加至少一种持续运动层：

- 雾气流动
- 尘粒漂浮
- 光束缓慢移动
- 胶片颗粒
- 暗角轻微呼吸
- 鼠标视差

这就是后续轻量 Three.js / Canvas 的主要职责。

## 7. 3D、WebGL 与氛围特效规范

### 7.1 使用定位

Three.js 不作为重型 3D 展厅使用，而是作为氛围层。

允许使用：

- 全屏透明 WebGL 雾气
- 光束 shader
- 浮尘粒子
- 鼠标视差
- 微弱景深感
- 复彩显影粒子

不建议一开始使用：

- 低质量佛像模型
- 可自由行走 3D 场景
- 复杂相机控制
- 炫技粒子爆炸
- 强烈 bloom
- 赛博扫描线

### 7.2 雾气规范

雾气应像空气，而不是烟雾特效。

要求：

- 慢。
- 稀薄。
- 层次分明。
- 不遮挡主体。
- 不抢文字。
- 偏冷灰或淡墨色。
- 金色光只作为极弱穿透。

### 7.3 粒子规范

粒子模拟石窟空气中的浮尘。

要求：

- 小。
- 慢。
- 数量可控。
- 不像星空。
- 不像魔法粒子。
- 不做彩色闪烁。

## 8. 交互规范

### 8.1 滚动

当前序章采用四段式步进滚动，这是正确方向。

规范：

- 每次滚轮进入一个明确叙事段。
- 不允许停在随机半透明文字状态。
- 视频进度与文案段落对应。
- 字幕切换必须：上一句淡出 → 空白 → 下一句淡入。
- 不允许两句主文案同时出现。

### 8.2 点击进入

最后一段出现“进入”按钮。

规范：

- 圆形按钮。
- 暗金描边。
- hover 时白色圆形从下方填充。
- 字色由白变深墨。
- 不使用普通矩形按钮。

### 8.3 控制按钮

右下角按钮保持圆形。

用途：

- 设置
- 声音
- 后续可加菜单

规范：

- 尺寸约 `50px`。
- 金色细描边。
- hover 白色填充。
- 图标变深色。
- 不出现文字按钮。

## 9. 文案规范

### 9.1 语气

文案应该像展览导语，不像说明书。

关键词：

- 简短
- 克制
- 有画面感
- 有历史感
- 不煽情
- 不过度玄学

当前四句是序章基准：

1. `1400年前，金陵栖霞山屹立起一座承载信仰与时间的舍利塔。`
2. `山岩之间，千佛静默，色彩隐入岁月。`
3. `如今，我们再次向石窟发问。`
4. `GrottoMind`

后续文案应延续这个节奏。

### 9.2 学术边界

涉及复彩、历史颜色、造像原貌时必须说明：

- 这是数字推演。
- 不是历史原貌的绝对复原。
- 不替代文物修复。
- 目的是帮助观众理解色彩、时间与风化。

这条是项目严谨性的底线。

## 10. 后续页面设计原则

后续点击进入后，不要做传统“第二页”。

应该是：

- 一个新的沉浸场景。
- 一个空间节点。
- 一个低干扰 UI 系统。
- 少量热点。
- 少量字幕。
- 必要时出现展签式信息。

推荐模块顺序：

1. `石窟入口`
2. `山岩显影`
3. `复彩实验室`
4. `问窟者`
5. `共创一龛`
6. `作品说明`

每个模块都应像一个展厅，而不是网页 section。

## 11. 禁止事项

后续开发必须避免：

- 普通卡片网格
- 大面积玻璃拟态
- 厚重阴影
- 霓虹渐变
- 赛博蓝紫
- 旅游宣传页式标题排版
- 信息过密
- 多字体混用
- Logo 随意变形
- 文案过长
- 同屏多个主标题
- 低质量 3D 佛像模型
- 粒子过度炫技
- 背景视频暂停后完全静止

## 12. 当前序章不可随意改动项

在没有明确要求前，以下内容视为“已定稿基准”：

- 左上角 Logo 位置与白色样式
- 中央文案字体与大致位置
- 四段式滚动结构
- 加载动画的仪式感
- 右下角圆形控制按钮
- 暗金描边体系
- 视频作为主视觉
- 雾气、颗粒、光影作为氛围层
- Persepolis 式的全屏沉浸方向

后续所有新增内容，都应该从当前序章“长出来”，而不是另开一套视觉系统。
